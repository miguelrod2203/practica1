git help para desplegar todos los comandos si lo requieres

git init si queremos relacionar los archivos con un repositorio que tenemos guardado en el computador usamos git init este comando lo usamos una solo ves por repositorio (cuando queremos vincular las carpetas de nuestro proyecto con git colocamos git init se debe hacer al inicio

git status -s para ver que archivos no tienen seguimiento tdos los que tengan (??) en la lista que se va a desplegar no tienen seguimiento

para hacer un segumiento debemos mandarlo al area local

git add para indicar que archivos queremos que se le realice seguimiento git add ____ nombre del archivo... luego para comprobar usamos de nuevo el comando git status -s y ahora cuando se despliegue la lista debe haber cambiado de (??) a (A) lo que significa que añadio correctamente el articulo al area temporal

en el area temporal se puede ir monitoreando el progreso

git commit -m se utliza para mandar algun archivo en especifico del area temporal a nuestro repositorio este comando es el que nos sirve para viajar en el tiempo, se debe desplegar una lista y los archivos seleccionados deben tener ahora la letra (M)

git add . manda todos los archivos al area temporal

git log --oneline este comando nos permite mediante una lista ver todos los commit que hemos realizado junto con un codigo identificador y el comentario que colocamos, nos sirve para viajar atraves de los commit

git reset --hard _____ si queremos volver a un commit (una parte de la historia antes de modificar ) colocamos este comando y el codigo del commit que deseas eliminar

git reset --hard _____ si queremos volver a un commit (una parte de la historia antes de modificar ) si eliminaste n commit pero lo uieres de regreso puedes usar este mismo comando para hacerlo, solo coloca el codigo que corresponde

git push para subirlo a la nube pero antes debes usar el comando git commit -s

entrar a github debes crear tu cuenta, luego te vas a repositorios y creas uno nuevo le colocas un nombre una descripcion y si quieres que sea publico o privado y lo creas

te van aparecer tres cajas de codigo... debes tomar solo de la segunda caja la primera linea de codigos y copiarlas en git y esto le va a indicar a git que ya va a estar trabajando de forma remota. osea va crearun idetificador de todos nuestros archivos o de el que le indiquemos. soo se coloca una vez

luego de que hagamos todos los commit -m correspondentes tomamos de la misma segunda caja la segunda linea de codigo y la colocamos en gites posible que tepida el usuario y contraseña

luego de esto si actualizas la pag van a aparecer los archivos que tenemos de forma local igual poras ver los commit que hubieras realizado con su codigo
